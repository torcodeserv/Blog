Desarrollo de aplicaciones web con ASP.NET Core MVC

Guía para iniciarte en el mundo del desarrollo de aplicaciones Web

Introducción

El desarrollo web moderno requiere frameworks robustos, escalables y seguros. ASP.NET Core se ha establecido como una de las opciones más potentes para la creación de aplicaciones web y APIs. Este blog técnico profundiza en ASP.NET Core MVC, proporcionando todo lo que necesitas para comenzar a desarrollar aplicaciones web profesionales con esta tecnología.

¿Qué es ASP.NET Core?

ASP.NET Core es un framework de desarrollo web de código abierto, multiplataforma y de alto rendimiento creado por Microsoft. A diferencia de su predecesor, ASP.NET Framework, ASP.NET Core está diseñado para ejecutarse no solo en Windows sino también en macOS y Linux.

Sus características principales incluyen:

  • Arquitectura modular y ligera
  • Soporte para múltiples plataformas
  • Alto rendimiento y escalabilidad
  • Inyección de dependencias integrada
  • Configuración unificada

Características principales

ASP.NET Core MVC proporciona una arquitectura basada en el patrón Modelo-Vista-Controlador, permitiendo una clara separación de responsabilidades en tu aplicación:

  • Modelos: Representan los datos de la aplicación y la lógica de negocio que determina cómo se pueden manipular.
  • Vistas: Muestran la interfaz de usuario y presentan los datos a los usuarios.
  • Controladores: Manejan las interacciones del usuario, trabajando con el modelo y seleccionando una vista para mostrar.

Además, ofrece:

  • Sistema de routing flexible
  • Motor de vistas Razor potente y fácil de aprender
  • Validación del lado del cliente y del servidor
  • API web integrada y soporte para JSON
  • Pruebas unitarias simplificadas

Ventajas y casos de uso

Entre las principales ventajas de utilizar ASP.NET Core MVC encontramos:

  • Alto rendimiento comparado con otros frameworks
  • Seguridad mejorada y actualizaciones regulares
  • Soporte para contenedores Docker
  • Excelente integración con Azure y otros servicios en la nube
  • Gran comunidad y documentación extensa

Los casos de uso ideales incluyen:

  • Aplicaciones empresariales complejas
  • Servicios RESTful y microservicios
  • Aplicaciones en la nube
  • Portales web que requieren alto rendimiento

Requisitos del sistema

Para comenzar a desarrollar con ASP.NET Core MVC necesitarás:

  • .NET SDK (versión 8.0 o superior recomendada)
  • Un IDE como Visual Studio, Visual Studio Code o JetBrains Rider
  • Conocimientos básicos de C# y HTML
  • Git para control de versiones (opcional pero recomendado)

Modelos (Models)

Creación del modelo de Productos

Archivo Proucto.cs

En ASP.NET MVC, el modelo representa la lógica de datos y la entidad de dominio de la aplicación. El modelo de producto generalmente se implementa como una clase C# que define las propiedades y características de un producto, como su nombre, descripción, precio, etc. Se utiliza para representar los datos de los productos en la aplicación, permitiendo que los controladores y las vistas interactúen con estos datos​

En la carpeta Models creamos un archivo de nombre Producto.cs

        using System;
        using System.ComponentModel.DataAnnotations;
        
        namespace ApplicacionWeb.Models
        {
            public class Producto
            {
                public int Id { get; set; }

                [Required(ErrorMessage = "El nombre es obligatorio.")]
                [StringLength(100, ErrorMessage = "El nombre no puede exceder los 100 caracteres.")]
                public string Nombre { get; set; }
        
                [StringLength(500, ErrorMessage = "La descripción no puede exceder los 500 caracteres.")]
                public string Descripcion { get; set; }
        
                [Required(ErrorMessage = "El precio es obligatorio.")]
                [Range(0.01, double.MaxValue, ErrorMessage = "El precio debe ser mayor a 0.")]
                public decimal Precio { get; set; }
        
                [Required(ErrorMessage = "El stock es obligatorio.")]
                [Range(0, int.MaxValue, ErrorMessage = "El stock no puede ser negativo.")]
                public int Stock { get; set; }
        
                public DateTime? FechaCaducidad { get; set; }
            }
        }
                            
        
Explicación:

        [Required]:
            •	Se utiliza para marcar los campos obligatorios (Nombre, Precio, Stock).
            •	Incluye un mensaje de error personalizado para mejorar la experiencia del usuario.
        [StringLength]:
            •	Limita la longitud máxima de los campos de texto (Nombre y Descripcion).
        [Range]:
            •	Valida que el Precio sea mayor a 0 y que el Stock no sea negativo.
        FechaCaducidad:
            •	Es opcional, por lo que se define como DateTime? (tipo nullable).
        Espacio de nombres (namespace):
            •	El modelo se coloca en la carpeta Models dentro del proyecto para mantener una estructura organizada.

Controladores (Controllers)

Creación del Controlador de Productos

Crea un controlador llamado ProductosController que manejará las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para el modelo Producto.

En la carpeta Controllers creamos un archivo de nombre ProductosController.cs

using Microsoft.AspNetCore.Mvc; using ApplicacionWeb.Models; namespace ApplicacionWeb.Controllers { public class ProductosController : Controller { private static List<producto> _productos = new List<Producto> { new Producto { Nombre = "Producto 1", Precio = 10.5m, Stock = 20 }, new Producto { Nombre = "Producto 2", Precio = 15.0m, Stock = 10 } }; // GET: Producto public IActionResult Index() { return View(_productos); } // GET: Producto/Create public IActionResult Create() { return View(); } // POST: Producto/Create [HttpPost] [ValidateAntiForgeryToken] public IActionResult Create(Producto producto) { if (ModelState.IsValid) { _productos.Add(producto); return RedirectToAction(nameof(Index)); } return View(producto); } // GET: Producto/Edit/5 public IActionResult Edit(int id) { var producto = _productos.ElementAtOrDefault(id); if (producto == null) { return NotFound(); } ViewBag.Id = id; return View(producto); } // POST: Producto/Edit/5 [HttpPost] [ValidateAntiForgeryToken] public IActionResult Edit(int id, Producto producto) { if (ModelState.IsValid) { _productos[id] = producto; return RedirectToAction(nameof(Index)); } return View(producto); } // GET: Producto/Delete/5 public IActionResult Delete(int id) { var producto = _productos.ElementAtOrDefault(id); if (producto == null) { return NotFound(); } return View(producto); } // POST: Producto/Delete/5 [HttpPost, ActionName("Delete")] [ValidateAntiForgeryToken] public IActionResult DeleteConfirmed(int id) { if (id >= 0 && id < _productos.Count) { _productos.RemoveAt(id); } return RedirectToAction(nameof(Index)); } } }

    Migraciones

    Las migraciones son una forma de mantener la estructura de la base de datos sincronizada con el modelo de datos de una aplicación. Permiten crear, modificar y eliminar tablas y columnas en la base de datos a medida que cambian los requisitos del modelo. Entity Framework Core (EF Core) proporciona un sistema de migraciones que facilita este proceso.

    • Cada cambio en el modelo de datos se registra como una migración, permitiendo un historial de modificaciones en la base de datos
    • Las migraciones pueden ser aplicadas a la base de datos mediante comandos de la línea de comandos o desde el código
    • Las migraciones permiten revertir cambios en la base de datos, facilitando la gestión de versiones
    • Las migraciones pueden ser generadas automáticamente a partir de cambios en el modelo de datos, lo que ahorra tiempo y esfuerzo
    • Las migraciones pueden ser personalizadas para adaptarse a requisitos específicos, como la creación de índices o la modificación de datos existentes
    • Las migraciones son una herramienta poderosa para mantener la integridad y la consistencia de la base de datos a lo largo del ciclo de vida de una aplicación

    Preparando el Entorno

    Los Paquetes NuGet

    Los paquetes NuGet son una forma de distribuir y compartir bibliotecas y herramientas para .NET. En este caso, se utilizan paquetes NuGet para trabajar con PostgreSQL y Entity Framework Core (EF Core). Estos paquetes proporcionan las bibliotecas necesarias para interactuar con la base de datos y realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en las entidades del modelo.

    Para trabajar con la Base de Datos PostgreSQL en ASP.NET Core, es necesario instalar los siguientes paquetes NuGet

    • Npgsql.EntityFrameworkCore.PostgreSQL - sirve como el proveedor de base de datos para Entity Framework Core (EF Core) cuando se utiliza con PostgreSQL.
    • Microsoft.EntityFrameworkCore.Tools - son una serie de herramientas de línea de comandos (CLI) y extensión de Visual Studio que simplifican el desarrollo de aplicaciones que usan Entity Framework Core (EF Core).
    • Microsoft.EntityFrameworkCore.Design - facilita la creación de modelos de base de datos y la gestión de migraciones durante el desarrollo

    La cadena de Conexión

    La cadena de conexión es una cadena de texto que contiene la información necesaria para conectarse a una base de datos. Incluye detalles como el nombre del servidor, el nombre de la base de datos, el usuario y la contraseña. En el caso de PostgreSQL, la cadena de conexión tiene un formato específico que incluye el nombre del host, el puerto, el nombre de la base de datos, el usuario y la contraseña. La cadena de conexión se utiliza en la configuración de la aplicación para establecer la conexión con la base de datos. En este caso, la cadena de conexión se almacena en el archivo appsettings.json y se utiliza en el archivo Program.cs para configurar el contexto de la base de datos.

    El Contexto de la Base de Datos

    Es una clase que representa una sesión con la base de datos y permite realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en las entidades del modelo. En este caso, el contexto de la base de datos se llama ApplicationDbContext y hereda de DbContext, que es la clase principal de Entity Framework Core (EF Core).

    Archivo ApplicationDbContext.cs

    Creamos una carpeta de nombre Data y creamos un archivo de nombre ApplicationDbContext.cs

    using Microsoft.EntityFrameworkCore; using ApplicacionWeb.Models; namespace ApplicacionWeb.Data { public class ApplicationDbContext : DbContext { public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options) { } public DbSet<Producto> Productos { get; set; } } }
    Entity Framework Core (EF Core)

    Es un mapeador objeto-relacional (ORM) de código abierto, multiplataforma y ligero, diseñado para trabajar con bases de datos desde aplicaciones .NET. Permite a los desarrolladores interactuar con bases de datos usando objetos .NET , simplificando el código de acceso a datos y permitiendo una mayor flexibilidad en la gestión de datos.

    DbContext

    Sirve como puente entre una aplicación ASP.NET Core y la base de datos. La clase ApplicationDbContext hereda de DbContext, que es la clase principal de Entity Framework Core. Esta clase representa una sesión con la base de datos y puede ser usada para consultar y guardar instancias de tus entidades.

    DbSet

    Es una propiedad dentro de una clase que extiende de DbContext en Entity Framework Core. Esto permite que Entity Framework Core sepa qué tablas debe crear en la base de datos. El nombre "Productos" será el nombre de la tabla en la Base de Datos PostgreSQL.

    El Modelo de Datos

    Es una representación de los datos que se almacenan en la base de datos. En este caso, el modelo de datos se representa mediante la clase Producto, que contiene las propiedades que corresponden a las columnas de la tabla en la base de datos. Entity Framework Core (EF Core) utiliza este modelo para mapear los datos entre la base de datos y la aplicación. El modelo de datos se define como una clase que hereda de DbContext y contiene propiedades que representan las tablas de la base de datos. Cada propiedad de la clase representa una tabla en la base de datos y cada propiedad dentro de la clase representa una columna en esa tabla. Entity Framework Core (EF Core) utiliza este modelo para mapear los datos entre la base de datos y la aplicación. En este caso, la clase Producto representa una tabla en la base de datos que contiene información sobre los productos. Cada propiedad de la clase representa una columna en esa tabla y cada instancia de la clase representa una fila en la tabla.
